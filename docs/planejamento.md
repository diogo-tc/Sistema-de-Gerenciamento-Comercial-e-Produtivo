# Planejamento das Sprints

## Metodologia de Desenvolvimento

O projeto será desenvolvido utilizando metodologia ágil com sprints quinzenais, permitindo entregas parciais e apresentações intermediárias ao longo do semestre.

A equipe utilizará:

- GitHub para versionamento e organização do projeto;
- Desenvolvimento incremental;
- Divisão de tarefas entre os integrantes;
- Reuniões periódicas para acompanhamento da evolução do sistema.

O sistema será desenvolvido como uma aplicação web voltada para uso do gerente da empresa, permitindo gerenciamento administrativo integrado com banco de dados relacional.

# Planejamento das Sprints

## Sprint 1 — Planejamento e Modelagem
**Duração:** Semanas 1 e 2

### Objetivos principais
- Definição dos requisitos do sistema;
- Levantamento das funcionalidades principais;
- Organização do repositório GitHub;
- Definição da arquitetura inicial;
- Criação dos diagramas UML;
- Modelagem inicial do banco de dados;
- Definição das tecnologias do projeto.

### Entregas esperadas
- README inicial;
- Diagramas UML;
- Modelo entidade-relacionamento;
- Estrutura inicial do projeto no GitHub.

---

## Sprint 2 — Estrutura Base do Sistema
**Duração:** Semanas 3 e 4

### Objetivos principais
- Configuração do ambiente de desenvolvimento;
- Criação da estrutura Flask;
- Integração inicial com MySQL;
- Implementação do sistema de login;
- Estruturação do frontend com Bootstrap;
- Criação da navegação principal do sistema.

### Entregas esperadas
- Sistema web funcionando localmente;
- Login funcional;
- Integração inicial com banco de dados.

---

## Sprint 3 — Cadastro de Clientes e Fornecedores
**Duração:** Semanas 5 e 6

### Objetivos principais
- Implementação do CRUD de clientes;
- Implementação do CRUD de fornecedores;
- Integração completa dessas entidades com banco de dados;
- Criação de formulários e telas administrativas;
- Testes das funcionalidades implementadas.

### Entregas esperadas
- Cadastro, edição, exclusão e consulta de clientes;
- Cadastro, edição, exclusão e consulta de fornecedores.

---

## Sprint 4 — Controle de Estoque e Funcionários
**Duração:** Semanas 7 e 8

### Objetivos principais
- Desenvolvimento do módulo de estoque;
- Controle de entradas e saídas;
- Cadastro de funcionários;
- Relacionamento entre tabelas no banco de dados;
- Melhorias visuais no frontend;
- Testes de integração.

### Entregas esperadas
- Controle básico de estoque funcional;
- Cadastro de funcionários integrado ao sistema.

---

## Sprint 5 — Manutenção, Ajustes e Testes
**Duração:** Semanas 9 e 10

### Objetivos principais
- Implementação do módulo de manutenção de máquinas;
- Correções gerais do sistema;
- Refatoração de código;
- Testes funcionais;
- Organização do repositório;
- Atualização da documentação técnica.

### Entregas esperadas
- Sistema praticamente completo;
- Funcionalidades integradas;
- Testes realizados.

---

## Sprint 6 — Finalização do Projeto
**Duração:** Semanas 11 e 12

### Objetivos principais
- Correções finais;
- Melhorias de usabilidade;
- Revisão da documentação;
- Revisão dos diagramas;
- Preparação da apresentação final;
- Organização definitiva do GitHub.

### Entregas esperadas
- Sistema final funcionando;
- Documentação completa;
- Projeto pronto para apresentação.
